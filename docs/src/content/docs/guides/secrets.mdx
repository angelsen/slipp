slipp integrates with Ansible Vault for secure secret management. Generate secrets, store them encrypted, and use them in deployments and local development.

## Quick Start

<Steps>

1. **Generate a secret**

   ```bash
   slipp secret
   # Output: 64 hex chars (256-bit)
   ```

2. **Add to vault**

   ```bash
   slipp secrets add vault_db_password myproject
   ```

3. **Use in templates**

   ```yaml title="group_vars/all.yml"
   database_password: "{{ vault_db_password }}"
   ```

4. **Encrypt vault**

   ```bash
   ansible-vault encrypt inventory/host_vars/myhost/vault.yml
   ```

</Steps>

## Generating Secrets

### Basic secret

```bash
slipp secret
```

Output: 64 hex characters (256-bit entropy)

### Options

<Tabs>
<TabItem label="Length">
```bash
# 16 bytes (128-bit)
slipp secret --bytes 16

# 64 bytes (512-bit)
slipp secret --bytes 64
```
</TabItem>
<TabItem label="Encoding">
```bash
# Base64 encoded
slipp secret --base64

# ULID format
slipp secret --ulid
```
</TabItem>
<TabItem label="JWK">
```bash
# RSA 2048-bit keypair
slipp secret --jwk

# RSA 4096-bit keypair
slipp secret --jwk --bits 4096
```
</TabItem>
</Tabs>

## Managing Vaults

### List available vaults

```bash
slipp secrets list
```

Shows all registered projects with vault files:

```
Available vaults:
project      vault                    secrets
myproject    inventory/.../vault.yml  5
auth-service inventory/.../vault.yml  3
```

### List secrets in a vault

```bash
slipp secrets list myproject
```

```
Secrets in inventory/host_vars/myhost/vault.yml:
  - vault_db_password
  - vault_api_key
  - vault_jwt_secret
```

### Get template string

```bash
slipp secrets list myproject vault_db_password
# Output: {{ vault_db_password }}
```

## Adding Secrets

### Add random secret

```bash
slipp secrets add vault_db_password myproject
```

Generates a random secret and adds it to the vault.

### Add JWK keypair

```bash
slipp secrets add vault_signing_key myproject --jwk
```

Generates an RSA keypair for JWT signing.

### Custom entropy

```bash
slipp secrets add vault_session_key myproject --bytes 64
```

## Syncing Secrets

Auto-generate secrets for all `{{ vault_* }}` references:

```bash
slipp secrets sync inventory/host_vars/myhost/vars.yml
```

<Steps>

1. **Scans** the YAML file for `{{ vault_* }}` patterns
2. **Lists** found references
3. **Generates** random secrets for each
4. **Creates** `vault.yml` in the same directory

</Steps>

:::caution
After sync, encrypt the vault:
```bash
ansible-vault encrypt inventory/host_vars/myhost/vault.yml
```
:::

## Using Secrets

### In Ansible templates

Reference vault secrets in your vars files:

```yaml title="group_vars/all.yml"
database:
  password: "{{ vault_db_password }}"

api:
  secret_key: "{{ vault_api_key }}"
```

### In run profiles

Load vault secrets as environment variables:

```bash
slipp run dev --vault myproject --cmd "npm run dev"
```

Secrets are transformed and injected:
- `vault_db_password` becomes `DB_PASSWORD`
- `vault_api_key` becomes `API_KEY`

### Multiple vaults

Combine secrets from multiple projects:

```bash
slipp run dev --vault myproject --vault shared-secrets
```

## Vault Workflow

### Initial setup

<Steps>

1. **Create vars file with references**

   ```yaml title="inventory/host_vars/myhost/vars.yml"
   matrix_sliding_sync_shared_secret: "{{ vault_sliding_sync_secret }}"
   matrix_postgres_connection_password: "{{ vault_postgres_password }}"
   ```

2. **Generate secrets**

   ```bash
   slipp secrets sync inventory/host_vars/myhost/vars.yml
   ```

3. **Encrypt vault**

   ```bash
   ansible-vault encrypt inventory/host_vars/myhost/vault.yml
   ```

4. **Deploy**

   ```bash
   slipp deploy
   # Prompts for vault password
   ```

</Steps>

### Adding new secrets

<Steps>

1. **Add to vault**

   ```bash
   slipp secrets add vault_new_secret myproject
   ```

2. **Reference in vars**

   ```yaml
   new_service_key: "{{ vault_new_secret }}"
   ```

3. **Redeploy**

   ```bash
   slipp deploy
   ```

</Steps>

## Pulling from External Sources

Pull credentials from external services via browser-based approval.

### How It Works

<Steps>

1. **Start pull**

   ```bash
   slipp secrets pull nor-auth myproject
   ```

2. **Browser opens** for authentication and resource selection

3. **Approve** the credential export in your browser

4. **Credentials stored** encrypted in your vault

</Steps>

### nor-auth Integration

Pull bot credentials and API keys from nor-auth console:

```bash
slipp secrets pull nor-auth
```

This exports:
- **Bot credentials** → `vault_nor_bot_<name>_*` variables
- **API keys** → `vault_nor_api_key_<name>` variables

### Options

| Option | Default | Description |
|--------|---------|-------------|
| `--timeout` | 300s | Time to wait for browser approval |

### Troubleshooting

:::caution[Browser didn't open?]
Copy the URL from terminal output and open manually.
:::

:::caution[Timeout expired?]
Increase timeout: `slipp secrets pull nor-auth --timeout 600`
:::

:::caution[Connection refused?]
Ensure no firewall blocks localhost callback port (49152-65535 range).
:::

## Security Best Practices

:::caution[Never commit unencrypted vaults]
Always encrypt vault files before committing:
```bash
ansible-vault encrypt vault.yml
```
:::

:::tip[Use .gitignore]
Add unencrypted vault files to `.gitignore`:
```
# Unencrypted vaults (temporary)
**/vault.yml.unencrypted
```
:::

:::tip[Vault password management]
Store vault password securely:
- Use a password manager
- Or set `ANSIBLE_VAULT_PASSWORD_FILE` environment variable
:::

## Troubleshooting

:::caution[Vault password incorrect?]
Check you're using the correct password. Re-encrypt if needed:
```bash
ansible-vault rekey vault.yml
```
:::

:::caution[Secret not found?]
- Verify vault path: `slipp secrets list`
- Check secret name matches: `slipp secrets list myproject`
- Ensure vault is encrypted: `file vault.yml` should show "ASCII text"
:::
